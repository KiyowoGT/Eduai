# Use Case: Deep Feedback (AI-Graded Feedback)

Actor: System / AI / Tutor

Description: Setelah kuis selesai, sistem menghasilkan penilaian mendalam (rationale, reference pages, remediation suggestions) menggunakan LLM berdasarkan jawaban dan dokumen sumber.

Preconditions:
- Hasil kuis (per-question answers) tersimpan di DB.
- Dokumen referensi tersedia dan telah dianalisis (context).

Postconditions:
- Deep feedback tersimpan, diikat ke result_id, dan dapat diakses student/teacher.
- Jika feedback autogenerated, status disimpan (ready/failed).

Main Flow:
1. Setelah POST /api/results (quiz submission), server men-trigger generate_deep_feedback(resultId).
2. generate_deep_feedback mengumpulkan context (quiz items, document concepts, student answers) dan memanggil _call_groq/_call_gemini.
3. Hasil berupa: per-question rationale, overall summary, recommended学习 materials; disimpan sebagai deep_feedback record.
4. Notifikasi event sent to useRealtimeSocket (type: result_feedback_ready).

Alternative Flows:
- 2a. LLM timeouts: queue task for retry and mark feedback pending.
- 3a. Insufficient context: produce short feedback and flag for human review.

Mapping ke kode:
- Backend endpoints: POST /api/feedback/deep, GET /api/feedback/{resultId}
- Frontend: components/DocumentResult.jsx, pages/QuizResult.jsx
- AI: use _call_groq for pedagogical feedback; _call_gemini may be used to synthesize document-based references.
